LogWrapper - version 1.2
LogWrapper utilise la réflection pour déterminer si les logs doivent être
envoyés à Log4j 2 ou dans la console.

Certaines propriétés peuvent influer sur le comportement de LogWrapper.
LogWrapper.silentlySearchLoggingTools réglé sur true permet de ne pas afficher
les messages d'erreurs dûs à l'absence d'un outil de logging.
LogWrapper.ignoreLog4j2 réglé sur true permet de ne jamais utiliser le wrapper
de Log4j 2.
LogWrapper.consoleLogger.level peut être réglé sur "off", "fatal", "error",
"warn", "info", "trace", "debug" ou "all" pour modifier le niveau de trace du
logger console qui est par défaut "all".
Si vous utilisez un compilateur java de version 1.7 ou inférieure, vous devez
écrire une interface "Supplier" pour remplacer "java.util.function.Supplier".

  Logger.java

  COPYING.txt
  COPYING-LESSER.txt
  lisezmoi.txt

Logger.java - classe générique mère de tous les wrappers.

LogManager.java - détermine le log wrapper à utiliser et produit les loggers.

Log4j2Wrapper.java - wrapper pour l'utilisation de Apache Log4j 2.

LogConsoleWrapper.java - wrapper pour l'affichage des logs dans la console.

Level.java - niveaux des logs.
